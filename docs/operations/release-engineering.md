# Release engineering & support matrix

- Status: Draft (Phase 11)
- Date: 2026-07-18

## Reproducible release

A release is produced by `.github/workflows/release.yml` on a `v*` tag, on
GitHub-hosted runners only (no external infra). Each release carries:

- **Edge binaries** (`gatewayd`, `lab-simulator`) built `--release` from the
  pinned toolchain (`rust-toolchain.toml`).
- **Control-plane** publish output (pinned SDK via `global.json`).
- **SBOM** (SPDX JSON) generated by the SBOM action.
- **Checksums** (`SHA256SUMS`) over every artifact.
- **Build provenance attestation** (`actions/attest-build-provenance`).
- Auto-generated release notes.

Determinism aids: pinned toolchains/SDK, committed lockfiles (`Cargo.lock`,
`pnpm-lock.yaml`, `packages.lock.json`), `Deterministic=true` for .NET.

## Signing (⛔ requires identities — OPEN)

Not yet implemented; requires real signing identities the owner must provide:

- **Windows**: Authenticode code-signing certificate → sign the Windows Service
  installer and updater.
- **macOS**: Apple Developer ID → sign + **notarize** the launchd daemon/app.
- **Driver packages**: ed25519 signing key custody + rotation (the verification
  side is implemented in `driver-runtime`).
- **Update metadata**: signed, with staged channels (stable/canary) and rollback.

Until identities exist, releases are unsigned pre-release artifacts for internal
use only.

## Support matrix

| Component | Supported | Notes |
|-----------|-----------|-------|
| Edge OS | Windows 11 (first production target), macOS (recent) | certified per analyzer/firmware/OS |
| Transports | serial (hardware-validated), TCP, watched files | serial backend behind `serialport-backend` feature |
| Protocols | ASTM E1381/E1394 (parse + link + assembly), HL7 v2 (parse + MLLP + original-mode ACK) | explicit profiles in `docs/protocols/` |
| Control plane | .NET 10 (LTS) API | in-memory store today; PostgreSQL/EF + OIDC OPEN |
| Drivers | signed, data-first; verified install + rollback | certification is a clinical gate |

A profile may be technically tested yet remain **experimental** until controlled
expected/actual validation is signed off by authorized laboratory staff.

## Rollback

- **Drivers**: `driver-runtime` registry keeps version history; `rollback()`
  reverts to the prior version; revocation blocks bad keys/digests/drivers.
- **Delivery**: the durable outbox is idempotent (dedup on enqueue), so replays
  after rollback/restart never double-deliver.
- **Releases**: prior tagged artifacts + checksums are retained; redeploy the
  previous version. Staging rollback is exercised before any production change.

## Pre-release gate (every release)

Formatting, lint, type, unit, integration, contract, conformance, secret,
dependency, and license checks pass (CI + Security workflows). Parser code carries
property/fuzz coverage. Windows/macOS packaging smoke tests and signed-driver
promotion require manual approval.
